# 网格交易策略 - 完备逻辑

## 核心概念

1. **价格格子**：把价格区间分成多层，每层一个价格
2. **格子方向**：每个格子要么是买单，要么是卖单
3. **持仓均价**：网格是基于持仓均价建立的

## 正常运行时逻辑

### 初始化（init_grid）
- 输入：持仓均价 entry_price
- 规则：
  - 格子价格 > entry_price → 挂卖单（价格下跌时卖出）
  - 格子价格 < entry_price → 挂买单（价格上涨时买入）
- 记录：每个格子的 original_side（原始方向）

### 成交后（update_grid_after_fill）
- 买单成交 → 在下一格（更低价格）挂卖单
- 卖单成交 → 在下一格（更高价格）挂买单

### 动态调整（_adjust_grid_direction）
- 价格连续上涨时，买单自动变成卖单
- 价格连续下跌时，卖单自动变成买单

## 崩溃恢复场景

### 场景分析

假设：均价 $600，网格 $500-$700

**场景1：价格从 $620 涨到 $650 后回落**

| 格子 | 原始方向 | 价格 | 当前价格$650 | OKX状态 | 恢复后动作 |
|------|----------|------|--------------|----------|------------|
| $700 | sell | 700 > 600 | 700 < 650 | 卖单成交 | 补买单($600+) |
| $650 | sell | 650 > 600 | 650 = 650 | 卖单成交 | 补买单 |
| $600 | sell | 600 = 600 | 600 < 650 | 买单成交 | 补卖单 |
| $550 | buy | 550 < 600 | 550 < 650 | 买单持有 | 保持 |
| $500 | buy | 500 < 600 | 500 < 650 | 买单持有 | 保持 |

### 完备恢复算法

对于每个格子 `p_str`：

1. **使用 original_side 反查订单**
   - 生成 client_oid: `g{sym}{side_char}{index}t{fill_count}`
   - 注意：使用 original_side，不是当前 side！

2. **如果找到订单**
   - 已成交 (state=filled) → 标记 done，返回成交记录
   - 已取消 (state=canceled) → 重置 order_id=None

3. **如果没找到订单**
   - 根据 original_side 和当前价格判断：
     - original_side=sell 且 p < current_price → **卖单成交了** → 方向改为买，返回成交记录
     - original_side=buy 且 p > current_price → **买单成交了** → 方向改为卖，返回成交记录
     - 否则 → 方向不变，根据当前价格微调

4. **返回值**
   - 返回所有成交订单列表，供 update_grid_after_fill 处理
